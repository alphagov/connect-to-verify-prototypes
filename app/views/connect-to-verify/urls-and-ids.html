Submit URLs

You must submit URLs for your:

  • service start page
  • service entity id
  • Verify SAML response
  • Matching Service Adapter URL for matching

If you want to allow users to create new accounts for your service you must also submit a Matching Service Adapter URL for user account creation.

The Verify test environment will use these URLs to connect to specific parts of your service.

What is this?

Your service start page URL is the URL that gives users access to your service. For example, https://gov.uk/your-service-name

What is this?

Your service entity id is a URL that Verify will use to uniquely identify your service. For example https://gds-verify/connect-to-verify/test.

What is this?

The Verify SAML response URL is the URL you use to receive SAML responses from Verify. For example, https://{your-service-name}.{your-integration-domain}/verify/response

What is this?

Your matching service adapter matching URL is the URL used to receive Verify matching requests. For example, https://{your-integration-domain}/matching-service/POST

What is this?

Your matching service adapter user account creation URL is the URL used to receive Verify account create requests. For example, https://{your-integration-domain}/unknown-user-attribute-query
