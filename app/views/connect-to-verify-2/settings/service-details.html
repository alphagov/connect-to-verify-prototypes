Service details

Verify only accepts requests from services it knows about for security reasons.

To configure your service you need to tell Verify an entity id so we can uniquely identify your service and the URL we should send the user back to once their identity has been verified.

What is this?

Your entity id is a URL that Verify will use to uniquely identify your service. For example:

https://{your-service-name}.{your-integration-domain}/verify

What is this?

The SAML response URL is the URL you use to receive SAML responses from Verify. For example:

https://{your-service-name}.{your-integration-domain}/verify/response
